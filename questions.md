# Какова основная цель валидации пользовательского ввода в программном обеспечении?
* Валидация предотвращает ошибки выполнения из-за несоответствия типов или диапазонов данных, защищает систему от некорректных или вредоносных данных
# Для чего используется конструкция try...except в Python? Объясните ее назначение.
* используется для обработки исключений, возникающих во время выполнения программы. Блок try содержит код, который может вызвать ошибку, а блок except — код, который выполняется при возникновении указанного исключения. Это позволяет программе продолжить работу, а не аварийно завершаться при ошибке.
# В чем заключается разница между блоками else и finally в конструкции try...except?
* Блок else выполняется только в том случае, если в блоке try не возникло исключений.
* Блок finally выполняется всегда, независимо от того, было исключение или нет (даже при return, break, continue). Он предназначен для гарантированного освобождения ресурсов (закрытие файлов, соединений).
# Какое исключение (Exception) чаще всего возникает при попытке преобразовать нечисловую строку в целое число с помощью функции int()?
* При попытке преобразовать нечисловую строку в целое число с помощью int() чаще всего возникает исключение ValueError.
# Как организовать бесконечный цикл запроса ввода до тех пор, пока пользователь не предоставит корректные данные? Какой оператор прерывает этот цикл?
* Для организации бесконечного цикла запроса ввода до получения корректных данных используется конструкция while True. Внутри цикла запрашивается ввод, выполняется проверка, и при успешной валидации цикл прерывается оператором break.
# Для чего применяются регулярные выражения в контексте валидации ввода?
* Регулярные выражения применяются в контексте валидации ввода для проверки строк на соответствие сложным шаблонам (например, формат email, пароль с определёнными требованиями, номер телефона). Они позволяют гибко описывать допустимые структуры данных и автоматизировать проверку.
